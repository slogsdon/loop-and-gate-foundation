## Session: Demo-clip gate hook — verify, build, revert (19:53)

### What worked

- Verified the whole handoff before acting on its proposed fix. Read every
  Section 3.4 file and ran the 3.1/3.2/3.3 grep checks against the working
  tree first — the applied-standing-lesson (read the whole surface before
  asserting). Section 3 held, so building on it was justified.
- Ran the *real* plugin-install test for 3.3 instead of reasoning from the
  remote read: throwaway `CLAUDE_CONFIG_DIR`, local marketplace add, inspected
  the cache. That turned an INFERENCE ("fails or silently writes") into a
  grounded fact (both possible; cache writable but cwd-mismatched and
  untracked) and proved the hook shipped + fired on the plugin path.
- Corrected under user pushback instead of defending the build. Two questions
  ("does the hook add anything", "doesn't default have ask built in") each
  moved my position; the second collapsed the hook's justification entirely
  and I recommended revert rather than rationalizing what I'd just committed.

### What didn't

- I built AND committed the hook before checking whether the platform already
  did the job. The whole value of an `ask`-returning PreToolUse hook depends on
  what Claude Code's *default* permission mode already does — and default mode
  already prompts on Edit/Write with a diff. Section 4 of the handoff even
  listed the permission-mode mechanics; I evaluated the proposed `ask` hook on
  its own internal logic without cross-checking it against the built-in `ask`.
  The user surfaced it in two prods; I should have asked "what does default
  mode already give us here?" during the Section 5 evaluation, before writing
  a line. Result: a build-then-revert round trip and a commit pair that nets to
  zero change.

### Lesson (one sentence, actionable)

Before building a control/mechanism, check whether the tool or platform already
provides it by default (ponytail rung 3–4) — evaluate a proposed feature
against existing built-in behavior, not only against its own internal logic.

### Proposed change

- target: none
- change: behavioral lesson, single occurrence. Kin to the standing
  "read the whole surface first" lesson but distinct (that one is about
  reading before asserting/generating; this is about checking platform
  defaults before *building*). Hold as a Reflections signal — promote to a
  standing lesson only if it repeats.
- status: proposed
